Introduction to HTML

Beginner

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure and content of a web page.

What is HTML?

  • HTMLstands for HyperText Markup Language
  • HTML is the standard markup language for Web pages
  • HTML describes the structure of a Web page
  • HTML consists of a series of elements
  • HTML elements tell the browser how to display content

Did you know?

HTML was invented by Tim Berners-Lee in 1991. The latest version is HTML5, which added many new features like video, audio, and canvas elements !

A Simple HTML Document

HTML 
<!DOCTYPE html>
<html>
<head>
<title>
Page Title</title>
</head>
<body>
<h1>
My First Heading</h1>
<p>
My first paragraph.</p>
</body>
</html>

Explanation:

  • <!DOCTYPE html> - Defines this as an HTML5 document
  • <html> - The root element
  • <head> - Contains meta information
  • <title> - Specifies page title
  • <body> - Contains visible page content
  • <h1> - Defines a large heading
  • <p> - Defines a paragraph

HTML Basic Structure

Understanding HTML Document Structure

Every HTML document follows a standard structure that browsers use to display web pages correctly. Understanding this structure is essential for creating valid and well-formed HTML documents.

The Basic HTML Template

A complete HTML document consists of several key components that work together. Here's the basic structure every HTML page should have:

HTML 
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>My First Webpage</title>
  </head>
  <body>
    <h1>Welcome to HTML</h1>
    <p>This is my first webpage!</p>
  </body>
</html>

1. DOCTYPE Declaration

The <!DOCTYPE html> declaration is the very first line of an HTML document. It tells the browser that this is an HTML5 document and ensures the page is rendered in standards mode.

Important: Always include the DOCTYPE declaration at the beginning of your HTML files. Without it, browsers may render your page in "quirks mode," causing inconsistent behavior.

2. The HTML Element

The <html> element is the root element that contains all other HTML elements. The lang attribute specifies the language of the document (e.g., "en" for English), which helps search engines and screen readers.

HTML 
<html lang="en">
  <!-- All content goes here -->
</html>

3. The Head Section

The <head> element contains meta-information about the document that isn't displayed on the page. This includes the page title, character encoding, stylesheets, and scripts.

Common elements in the head section:

  • <meta charset="UTF-8"> - Defines character encoding for proper text display
  • <meta name="viewport"> - Makes the page responsive on mobile devices
  • <title> - Sets the page title shown in browser tabs and search results
  • <link> - Links external resources like CSS files
  • <meta name="description"> - Provides a page description for SEO
HTML 
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Learn HTML basics">
  <title>HTML Tutorial</title>
  <link rel="stylesheet" href="styles.css">
</head>

4. The Body Section

The <body> element contains all the visible content of the webpage - text, images, links, videos, and everything users see and interact with.

HTML 
<body>
  <header>
    <h1>My Website</h1>
    <nav>
      <a href="#home">Home</a>
      <a href="#about">About</a>
    </nav>
  </header>
  
  <main>
    <h2>Welcome!</h2>
    <p>This is the main content area.</p>
  </main>
  
  <footer>
    <p>© 2026 My Website</p>
  </footer>
</body>

Complete Example

Here's a complete HTML document with all the essential components:

HTML 
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="A simple HTML page example">
    <title>Complete HTML Structure</title>
  </head>
  <body>
    <header>
      <h1>My First Website</h1>
    </header>
    
    <main>
      <article>
        <h2>About HTML</h2>
        <p>HTML is the standard markup language for creating web pages.</p>
      </article>
    </main>
    
    <footer>
      <p>Created with HTML5</p>
    </footer>
  </body>
</html>

Key Takeaways

  • Always start with <!DOCTYPE html> to declare HTML5
  • The <html> element wraps all content
  • The <head> contains meta-information (not visible)
  • The <body> contains all visible content
  • Proper indentation makes your code readable and maintainable
  • Every opening tag must have a corresponding closing tag
Pro Tip: Use proper indentation (2 or 4 spaces) to show the hierarchy of your HTML elements. This makes your code easier to read and debug!

HTML Elements

Beginner

What Are HTML Elements?

The foundation of HTML documents is HTML elements. They are used to specify a web page's layout and content. A start tag, content, and an end tag are present in every HTML element.

1. Start Tag

It identifies the element's type, whereas the end tag specifies where the element finishes.

For example, <html> is the start tag as it starts the HTML document.

2. Content

Text, photos, or other HTML components can all be used as an element's content. The visual and interactive components of a website are created by developers by mixing various elements.

For example,

HTML 
<p>
This is a paragraph.</p>

In this, the <p> tag starts the paragraph, which includes the content "This is a paragraph,"and closes the paragraph with the </p> tag.

3. End Tag

An end tag in an HTML element is the tag that marks the end of the element. The end tag has the same name as the start tag but with a forward slash (/) before the tag name.

For example, the end tag for the start tag <html> is </html> .

HTML Attributes

Beginner

Overview

An attribute in HTML is a special word within an element's opening tag that controls its behavior. HTML attributes are a modifier of an HTML element type. Attributes either modify a type's default functionality or provide functionality without which certain types cannot function correctly. Attributes are added to HTML start tags. An attribute usually consists of a name and value, like this: name="value".

What are Attributes in HTML?

HTML attributes are pieces of markup language that are used to modify the behavior or appearance of HTML elements. For Example: If you want to add links to your website you can use the hrefattribute, similarly if you want to change the height and width you can use the heightand widthattribute. HTML elements can also be customized by setting attributes such as color, size, and functionality.

Syntax:

Attributes can be set in the opening HTML tag as follows:

<tag-name attribute_name="value">
Content</tag-name>

Example:

<!DOCTYPE html>
<html>
<body>
<h2>
href Attribute Example</h2>
<!-- Here we are providing the link to be displayed when clicked using the href attribute -->
<a href="https://youtube.com"target="_blank">
Welcome to Youtube !</a>
</body>
</html>

HTML Canvas is used to draw graphics on a web page using JavaScript. It provides a space where developers can create shapes, images, animations, and games directly in the browser.

The canvas element itself is only a container for graphics, and the actual drawing is done using JavaScript.

  • game development
  • animations
  • image editing tools

Syntax

</canvas>

  • idis used to access the canvas using JavaScript.
  • widthand heightdefine the size of the canvas.

</canvas>

  • getElementById()selects the canvas element.
  • getContext("2d")creates a 2D drawing context.

SVG (Scalable Vector Graphics) is used to create graphics on a web page using XML-based code. Unlike Canvas, SVG graphics are resolution-independent and can be scaled without losing quality.

SVG graphics are defined using XML tags directly in HTML, making them easy to create, modify, and style with CSS.

SVG is commonly used for:

  • logos and icons
  • charts and diagrams
  • illustrations
  • scalable graphics for responsive design

<!-- SVG shapes go here -->
</svg>

Explanation

  • <svg> defines the SVG container.
  • widthand heightdefine the size of the SVG area.
  • SVG shapes are created using tags like <circle> , <rect> , <line> , etc.

Example - Circle

<svg width="200"height="200">
<circle cx="100"cy="100"r="50"fill="blue"/>
</svg>

Example - Rectangle

<svg width="300"height="200">
<rect x="50"y="50"width="150"height="100"fill="green"/>
</svg>

Explanation

  • <circle> creates a circle shape.
  • cxand cydefine the center position.
  • rdefines the radius.
  • <rect> creates a rectangle.
  • xand ydefine the starting position.
  • fillsets the color of the shape.

Example for a Tech Learning Website

HTML SVG Example</h2>

function drag(ev) {
    ev.dataTransfer.setData("text", ev.target.id);

}

</script>

HTML Block & Inline Elements

Every HTML element has a default display property that determines how it behaves on the webpage. Elements are classified into two main categories: Block Elementsand Inline Elements.

Block-level Elements

Block-level elements always start on a new line and take up the full width available, stretching from left to right as far as they can.

Visual Example: Block Elements

<!-- Each block element takes FULL WIDTH -->

<div style="background: lightblue; padding: 10px;">
?? Block 1: I take the full width </div>

<p style="background: lightgreen; padding: 10px;">
?? Block 2: I also take full width (new line) </p>

<h2 style="background: lightcoral; padding: 10px;">
?? Block 3: Me too ! Always starts on new line </h2>

List of Block Elements

  • <div> - Generic container
  • <p> - Paragraph
  • <h1> - <h6> - Headings
  • <ul> , <ol> , <li> - Lists
  • <header> , <footer> , <section> - Semantic elements
  • <form> , <table> - Interactive & data elements

Inline Elements

Inline elements do not start on a new line and only take up as much width as necessary. They flow with the content.

  • <a> - Links
  • <strong> , <b> 4t6ddd - Bold text
  • <em> , <i> - Italic text
  • <img> - Images
  • <input> , <button> - Form elements

Changing Display Property

You can change element behavior using CSS display property:

<!-- Example: Making navigation links display as blocks on mobile -->
<style>

/* Mobile: Stack links vertically (block) */
.nav-link {
    display: block;
    padding: 15px;
    background: #667eea;
    color: white;
    margin: 5px 0;
}

/* Desktop: Show links horizontally (inline-block) */
@media (min-width: 768px) {
    .nav-link {
        display: inline-block;
        margin: 0 10px;
    }
}

</style>

The HTML <div> Element

The <div> element is one of the most commonly used HTML elements. It stands for "division"and is used as a container to group other HTML elements together.

The <div> is a block-level element, which means it starts on a new line and takes up the full width available.

Basic Syntax

<div>
Content goes here... </div>

Why Use <div> ?

  • Grouping: Group related elements together
  • Styling: Apply CSS styles to multiple elements at once
  • Layout: Create page sections and layouts
  • JavaScript: Manipulate groups of elements

Div vs Semantic Tags

While <div> is very useful, HTML5 introduced semantic alternatives that better describe their content:

  • <header> instead of <div class="header">
  • <footer> instead of <div class="footer">
  • <section> instead of <div class="section">
  • <article> for independent content
  • <nav> for navigation menus

Best Practices for Divs ??

  • ? Use semantic tags first: < header> , <main> , <footer> better than divs
  • ??? Meaningful class names: "course-card"not "box1"
  • ?? Avoid "div soup": Don't nest too many divs unnecessarily
  • ?? Organize logically: Group related content together
  • ?? Style consistently: Use classes, not inline styles everywhere
  • ??? Reusable components: Create common patterns (cards, alerts)

Responsive Web Design

Responsive web design makes your website look good on all devices (desktops, tablets, and phones). It automatically adjusts the layout, images, and content based on the screen size.

Understanding the Viewport Tag ??

What it does:Controls how your website appears on mobile devices 

<meta name="viewport"content="width=device-width, initial-scale=1.0">

Breaking it down:

  • width=device-width?? Makes width match the device screen (not 980px)
  • initial-scale=1.0?? Sets zoom level to 100% (no zooming)
  • user-scalable=no?? (Optional) Prevents pinch-to-zoom

Result:Your website looks perfect on phones, tablets, and desktops ! ??

Responsive Units

Use relative units instead of fixed pixels for better responsiveness:

Unit Description Example
% Percentage of parent element width: 50%;
em Relative to font size padding: 2em;
rem Relative to root font size font-size: 1.5rem;
vw Viewport width (1vw=1%) width: 100vw;
vh Viewport height (1vh=1%) height: 100vh;
