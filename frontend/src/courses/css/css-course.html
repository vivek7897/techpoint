CSS Introduction

CSS (Cascading Style Sheets)is the language used to style and layout web pages. CSS describes how HTML elements should be displayed.

What is CSS?

  • CSS stands for Cascading Style Sheets
  • CSS describes how HTML elements are to be displayed on screen, paper, or in other media
  • CSS saves a lot of work by controlling the layout of multiple web pages at once
  • External stylesheets are stored in CSS files

Why Use CSS?

CSS is used to define styles for your web pages, including the design, layout and variations in display for different devices and screen sizes.

CSS Example:

body {
    background-color: lightblue;
}

h1 {
    color: white;
    text-align: center;
}

p {
    font-family: verdana;
    font-size: 20px;
}

CSS Syntax

A CSS rule consists of a selector and a declaration block:

selector {
    property: value;
    property: value;
}
💡 Tip:CSS can be added to HTML in three ways: Inline, Internal, and External stylesheets.

CSS Syntax

A CSS rule-set consists of a selector and a declaration block:

CSS Selector

The selector points to the HTML element you want to style.

Example:


/* Element Selector */
p {
    color: red;
}

/* Class Selector */
.myClass {
    color: blue;
}

/* ID Selector */
#myId {
    color: green;
}

CSS Declaration

Each declaration includes a CSS property name and a value, separated by a colon.

💡 Remember:Always end CSS declarations with a semicolon, and declaration blocks with curly braces.

CSS Selectors

CSS selectors are used to "find"(or select) the HTML elements you want to style.

Types of CSS Selectors

1. Element Selector

Selects HTML elements based on the element name.

p {
    text-align: center;
    color: red;
}

2. ID Selector

Uses the id attribute of an HTML element to select a specific element. 

#para1 {
    text-align: center;
    color: blue;
}

3. Class Selector

Selects HTML elements with a specific class attribute.

.center {
    text-align: center;
    color: green;
}

/* Specific element with class */
p.center {
    text-align: center;
}

4. Universal Selector

Selects all HTML elements on the page.

* {
    margin: 0;
    padding: 0;
}

5. Grouping Selector

Selects all the HTML elements with the same style definitions. 

h1,
h2,
p {
    text-align: center;
    color: red;
}

CSS Flexbox

CSS Flexbox (Flexible Box Layout) is a powerful layout model that makes it easy to design flexible and responsive layouts without using float or positioning. It's one of the most important modern CSS features !

Flexbox is perfect for creating navigation bars, card layouts, centering content, and building responsive designs that adapt to different screen sizes.

Understanding Flexbox Basics

Flexbox works with two types of elements:

  • Flex Container- The parent element (using display: flex)
  • Flex Items- The direct children of the flex container

CSS Grid

CSS Grid Layout is the most powerful layout system available in CSS. It's a two-dimensional layout system that handles both rows and columns at the same time, making it perfect for creating complex web layouts.

While Flexbox is one-dimensional (either row OR column), Grid allows you to work with both dimensions simultaneously, making it ideal for page layouts, image galleries, and complex UI designs.

Understanding Grid Basics

CSS Grid works with:

  • Grid Container- The parent element (using display: grid)
  • Grid Items- The direct children of the grid container
  • Grid Lines- The dividing lines between rows and columns
  • Grid Tracks- The space between two grid lines (rows or columns)
  • Grid Cells- A single unit of the grid
  • Grid Areas- Rectangular space made up of one or more cells

Grid Container Properties

1. grid-template-columns

Defines the number and width of columns in your grid.

.container {
    /* 3 equal columns */
    grid-template-columns: 1fr 1fr 1fr;

    /* Same as above using repeat() */
    grid-template-columns: repeat(3, 1fr);

    /* Fixed and flexible columns */
    grid-template-columns: 200px 1fr 2fr;

    /* Mix of units */
    grid-template-columns: 100px auto 200px;

    /* Responsive columns (auto-fit) */
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
}

CSS Animations

CSS Animations allow you to create complex, multi-step animations without JavaScript. Unlike transitions that require a trigger (like hover), animations can run automatically, loop infinitely, or play once when page loads.

Animations are perfect for loading spinners, attention-grabbing effects, welcome animations, and creating engaging user experiences that bring your website to life !

Animations vs Transitions

Feature Transitions Animations
Trigger Needs a state change (hover, focus) Can run automatically
Steps 2 states (start and end) Multiple keyframes
Loop No Yes, can repeat infinitely
Control Limited Full control (pause, delay, direction)

Creating Animations - Two Steps

Step 1: Define @keyframes

First, create the animation using @keyframesrule. This defines what happens during the animation. 

@keyframes animationName {
    from {
        /* Starting styles */
        opacity: 0;
    }

    to {
        /* Ending styles */
        opacity: 1;
    }
}

/* Or use percentages for more control */
@keyframes animationName {
    0% {
        /* Start */
        transform: translateX(0);
    }

    50% {
        /* Middle */
        transform: translateX(100px);
    }

    100% {
        /* End */
        transform: translateX(0);
    }
}

Step 2: Apply Animation

Then, apply the animation to an element using the animationproperty. 

.element {
    animation: animationName 2s ease-in-out infinite;
}

Animation Properties

1. animation-name

Specifies which @keyframes animation to use.

.element {
    animation-name: slideIn;
}

2. animation-duration

How long the animation takes to complete one cycle.

.element {
    animation-duration: 2s;
    /* 2 seconds */
    animation-duration: 500ms;
    /* 500 milliseconds */
}

3. animation-timing-function

Controls the speed curve of the animation.

.element {
    animation-timing-function: ease;
    /* Default */
    animation-timing-function: linear;
    /* Constant speed */
    animation-timing-function: ease-in;
    /* Slow start */
    animation-timing-function: ease-out;
    /* Slow end */
    animation-timing-function: ease-in-out;
    /* Slow start and end */
}

4. animation-delay

Delays the start of the animation.

.element {
    animation-delay: 1s;
    /* Wait 1 second before starting */
    animation-delay: 0s;
    /* Start immediately (default) */
}

5. animation-iteration-count

How many times the animation should repeat.

.element {
    animation-iteration-count: 3;
    /* Play 3 times */
    animation-iteration-count: infinite;
    /* Loop forever */
    animation-iteration-count: 1;
    /* Play once (default) */
}

6. animation-direction

Controls the direction of the animation.

.element {
    animation-direction: normal;
    /* Forward (default) */
    animation-direction: reverse;
    /* Backward */
    animation-direction: alternate;
    /* Forward, then backward */
    animation-direction: alternate-reverse;
    /* Backward, then forward */
}

7. animation-fill-mode

Specifies styles applied before/after animation runs.

.element {
    animation-fill-mode: none;
    /* No styles applied (default) */
    animation-fill-mode: forwards;
    /* Keep final state */
    animation-fill-mode: backwards;
    /* Apply first keyframe before animation */
    animation-fill-mode: both;
    /* Both forwards and backwards */
}

8. animation-play-state

Pause or resume the animation.

.element {
    animation-play-state: running;
    /* Play (default) */
    animation-play-state: paused;
    /* Pause */
}

/* Pause on hover */
.element:hover {
    animation-play-state: paused;
}

9. animation (Shorthand)

Combines all animation properties in one line.



/* name duration timing-function delay iteration-count direction fill-mode */
.element {
    animation: slideIn 2s ease-in-out 0s infinite alternate both;

    /* Common shorthand */
    animation: slideIn 2s infinite;

    /* Multiple animations */
    animation: slideIn 2s, fadeIn 1s;
}

Animation Best Practices

✅ Do's:

  • Use transformand opacityfor better performance
  • Keep animations subtle and purposeful
  • Add animation-fill-mode: forwardsto maintain final state
  • Use will-changeproperty for complex animations
  • Test animations on different devices and browsers
  • Provide a way to disable animations (accessibility)

❌ Don'ts:

  • Don't animate width/height (use transform: scale()instead)
  • Don't overuse animations - they can be distracting
  • Don't make animations too fast (<0.2s) or too slow (> 5s)
  • Don't forget to consider users with motion sensitivity
  • Don't animate too many elements simultaneously

Accessibility Consideration

♿ Respect User Preferences:

Some users have motion sensitivity. Always respect their preferences: 



/* Disable animations for users who prefer reduced motion */
@media (prefers-reduced-motion: reduce) {
    * {
        animation-duration: 0.01ms !important;
        animation-iteration-count: 1 !important;
        transition-duration: 0.01ms !important;
    }
}

Performance Tips

⚡ Optimize Performance:

Fast Properties (GPU-accelerated):

  • transform- translate, scale, rotate, skew
  • opacity
  • filter(with caution)

Slow Properties (avoid animating):

  • width / height
  • top / left / right / bottom
  • margin / padding
  • border-width

Use will-change for complex animations: 

.animated-element {
    will-change: transform, opacity;
    animation: complexAnimation 2s infinite;
}

CSS Links

In CSS, the link styling is used to change the appearance of hyperlinks (< a> tags) on a webpage. By default, links appear blue with an underline, but CSS allows us to customize their color, decoration, and behavior when users interact with them.

CSS provides different link states to style links based on user interaction.

Link States in CSS

There are four important states of a link:

1. a:link

This represents a normal, unvisited link.

Example:

a:link {
    color: blue;
}

2. a:visited

This represents a link that the user has already visited.

Example:

a:visited {
    color: purple;
}

3. a:hover

This style is applied when the user moves the mouse over the link.

Example:

a:hover {
    color: red;
    text-decoration: underline;
}

4. a:active

This state appears when the user clicks the link.

Example:

a:active {
    color: green;
}

Important Note

The correct order for styling links is:

  1. a:link
  2. a:visited
  3. a:hover
  4. a:active

This order ensures that all link styles work properly.

Why is Display Property Important?

  • Controls element layout behavior
  • Determines how elements stack or sit side-by-side
  • Essential for creating responsive designs
  • Allows you to show or hide elements

Common Display Values

1. display: block

Block-level elements take up the full width available and always start on a new line.

Default block elements:< div> , < h1> -< h6> , < p> , < form> , < header> , < footer> , < section>

Example:

div {
    display: block;
    background-color: lightblue;
    padding: 10px;
    margin: 5px;
}

Inline elements only take up as much width as necessary and do not start on a new line. They flow with the text.

Default inline elements:< span> , < a> , < strong> , < em> , < img> , < button>

Example:

CSS Position

The positionproperty is used to control how and where an element is positioned on a webpage. It's one of the most powerful CSS properties for creating complex layouts and interactive designs.

Understanding positioning is essential for creating navigation bars, popups, tooltips, overlays, and many other modern web features.

The 5 Position Values

CSS provides 5 different position values:

  • static(default)
  • relative
  • absolute
  • fixed
  • sticky

Positioning Properties

Once you set a position value (except static), you can use these properties to fine-tune the element's location:

  • top- Distance from the top
  • right- Distance from the right
  • bottom- Distance from the bottom
  • left- Distance from the left
  • z-index- Stacking order (which element appears on top)

1. position: static

This is the defaultposition value. Elements are positioned according to the normal flow of the document. The top, right, bottom, left, and z-index properties have NO effecton static elements. 

div {
    position: static;
    /* Default - not necessary to specify */
}

2. position: relative

An element with position: relativeis positioned relative to its normal position. You can move it using top, right, bottom, and left properties, but the element's original space is preserved. 

.box {
    position: relative;
    top: 20px;
    /* Move 20px down from normal position */
    left: 30px;
    /* Move 30px right from normal position */
}

3. position: absolute

An element with position: absoluteis removed from the normal document flow and positioned relative to its nearest positioned ancestor(any parent with position: relative, absolute, or fixed). If no positioned ancestor exists, it positions relative to the document body. 

.container {
    position: relative;
    /* Create positioning context */
}

.box {
    position: absolute;
    top: 10px;
    /* 10px from top of container */
    right: 10px;
    /* 10px from right of container */
}

4. position: fixed

An element with position: fixedis positioned relative to the browser viewport. It stays in the same place even when the page is scrolled. 

.navbar {
    position: fixed;
    top: 0;
    left: 0;
    width: 100%;
    background: white;
    z-index: 1000;
    /* Ensure it stays on top */
}

Common Uses for Fixed Positioning:

  • Fixed Navigation Bars- Stay at top while scrolling
  • Floating Action Buttons- Always visible in corner
  • Chat Widgets- Fixed in bottom-right corner
  • Cookie Notices- Fixed at top or bottom

5. position: sticky

position: stickyis a hybrid between relative and fixed positioning. The element is treated as relative until it reaches a specified scroll position, then it becomes fixed. 

.section-header {
    position: sticky;
    top: 0;
    /* Stick to top when scrolling */
    background: white;
    z-index: 10;
}

How Sticky Works:

1. Scrolling Down:Element scrolls normally (relative behavior)
2. Reaches Top:Element "sticks"to top (fixed behavior)
3. Scrolling Back:Element returns to normal position

Perfect for:Table headers, section titles, navigation menus

.foreground {
    position: absolute;
    z-index: 10;
    /* In front */
}

.modal {
    position: fixed;
    z-index: 1000;

CSS Transitions

CSS Transitions allow you to change property values smoothly over a specified duration, creating smooth animations between different states. Instead of an instant change, transitions make elements transform gradually, creating a more polished and professional user experience.

Transitions are perfect for hover effects, button interactions, menu animations, and any time you want to add smooth visual feedback to user interactions.

Why Use Transitions?

  • Improve user experience with smooth visual feedback
  • Make interfaces feel more responsive and professional
  • Guide user attention to interactive elements
  • Add polish to your designs without complex code
  • Easy to implement - just add one CSS property !

Transition Properties

1. transition-property

Specifies which CSS properties should be transitioned.

.box {
    transition-property: background-color;
    /* Specific property */
    transition-property: width, height;
    /* Multiple properties */
    transition-property: all;
    /* All properties (default) */
}

2. transition-duration

Specifies how long the transition should take to complete.

.box {
    transition-duration: 0.3s;
    /* 300 milliseconds */
    transition-duration: 1s;
    /* 1 second */
    transition-duration: 500ms;
    /* 500 milliseconds */
}

3. transition-timing-function

Controls the speed curve of the transition effect.

.box {
    transition-timing-function: ease;
    /* Slow start, fast middle, slow end (default) */
    transition-timing-function: linear;
    /* Same speed throughout */
    transition-timing-function: ease-in;
    /* Slow start */
    transition-timing-function: ease-out;
    /* Slow end */
    transition-timing-function: ease-in-out;
    /* Slow start and end */
    transition-timing-function: cubic-bezier(0.68, -0.55, 0.27, 1.55);
    /* Custom */
}

4. transition-delay

Specifies when the transition should start.

.box {
    transition-delay: 0s;
    /* Start immediately (default) */
    transition-delay: 0.5s;
    /* Wait 0.5 seconds before starting */
    transition-delay: 200ms;
    /* Wait 200 milliseconds */
}

5. transition (Shorthand)

Combines all transition properties in one line.

.box {
    /* property duration timing-function delay */
    transition: all 0.3s ease 0s;

    /* Common shorthand (property and duration only) */
    transition: all 0.3s;

    /* Multiple transitions */
    transition: width 0.3s ease, height 0.5s ease-in;
}
💡 Most Common Usage:transition: all 0.3s ease; - This transitions all properties smoothly over 0.3 seconds !

Best Practices

✅ Do's:

  • Keep transitions short (0.2s - 0.5s) for better UX
  • Use easeor ease-outfor most cases
  • Transition specific properties when possible (better performance)
  • Add transitions to interactive elements (buttons, links, cards)
  • Test on different devices to ensure smooth performance

❌ Don'ts:

  • Don't make transitions too long (> 1s can feel sluggish)
  • Avoid transitioning too many properties simultaneously
  • Don't transition width/height on large elements (use transform: scale instead)
  • Don't overuse transitions - not every element needs animation
  • Avoid transitioning properties that cause layout recalculation

Performance Tips

⚡ Optimize Performance:

Fast Properties (GPU-accelerated):

  • transform- translate, scale, rotate
  • opacity

Slower Properties (avoid if possible):

  • width / height- causes layout recalculation
  • top / left / right / bottom- use transform: translate()instead
  • margin / padding

Example:Instead of left: 100px, use transform: translateX(100px)for better performance !

CSS Transform

The CSS transformproperty lets you rotate, scale, skew, or translate (move) an element. It modifies the coordinate space of the element, allowing you to create stunning visual effects without affecting the normal document flow.

Transform is one of the most powerful CSS properties and is essential for animations, hover effects, and creating dynamic, interactive designs. Best of all, transforms are GPU-accelerated, making them incredibly performant !

Why Use Transform?

  • 🚀 High Performance- GPU-accelerated, smooth animations
  • 🎨 No Layout Changes- Doesn't affect other elements
  • ✨ Stunning Effects- Rotate, scale, move, and skew elements
  • 🎯 Perfect for Interactions- Hover effects, animations
  • 📱 3D Transforms- Create depth and perspective

2D Transform Functions

1. translate() - Move Elements

Moves an element from its current position without affecting other elements. 


/* Move element */
transform: translate(50px, 100px);
/* x, y */
transform: translateX(50px);
/* Move horizontally */
transform: translateY(100px);
/* Move vertically */

/* Can use percentages (relative to element size) */
transform: translate(50%, 50%);

Responsive Design

Responsive web design ensures your website looks great and functions perfectly on all devices - from mobile phones to large desktop monitors. It's not just about making things smaller; it's about creating flexible, adaptive layouts that provide the best experience for every screen size.

In today's world, over 60% of web traffic comes from mobile devices. Responsive design is no longer optional - it's essential!

What is Responsive Design?

📱 Responsive Design Principles:

  • Fluid Grids- Layouts that scale proportionally
  • Flexible Images- Images that resize within containers
  • Media Queries- CSS rules for different screen sizes
  • Mobile-First- Design for mobile, then scale up
  • Touch-Friendly- Big buttons, easy navigation

Media Queries - The Core of Responsive Design

Media queries let you apply different CSS styles based on device characteristics like screen width, height, orientation, and more.

Basic Syntax



/* Mobile styles (default) */
.container {
    width: 100%;
    padding: 10px;

/* Tablet and up (768px and wider) */

        width: 750px;
        padding: 20px;
    }
}

/* Desktop and up (1024px and wider) */
@media (min-width: 1024px) {
    .container {
        width: 960px;
        padding: 30px;
    }
}

CSS Variables (Custom Properties)

CSS Variables, also known as CSS Custom Properties, allow you to store values that can be reused throughout your stylesheet. They make your CSS more maintainable, flexible, and dynamic. Think of them as containers that hold values you can use anywhere !

Unlike preprocessor variables (Sass, Less), CSS variables are live- they can be updated with JavaScript and cascade through the DOM !

Why Use CSS Variables?

✨ Benefits of CSS Variables:

  • 🎨 Easy Theming- Change colors across entire site instantly
  • ♻️ Reusability- Define once, use everywhere
  • 🔧 Dynamic- Update with JavaScript in real-time
  • 📦 Scoping- Can be global or local to elements
  • 🌊 Cascading- Inherit values from parent elements
  • 🚀 No Build Step- Pure CSS, no preprocessor needed

Basic Syntax

CSS variables are defined with --prefix and accessed with var()function: 



/* Define variables */
:root {
    --primary-color: #3498db;
    --secondary-color: #2ecc71;
    --font-size: 16px;
    --spacing: 20px;
}

/* Use variables */
.button {
    background-color: var(--primary-color);
    font-size: var(--font-size);
    padding: var(--spacing);
}

.heading {
    color: var(--primary-color);
    margin-bottom: var(--spacing);
}

Advanced CSS Selectors

Advanced CSS selectors give you powerful ways to target specific elements without adding extra classes or IDs. Master these selectors and you'll write cleaner, more maintainable CSS code!

These selectors let you target elements based on their attributes, position in the document, state, and relationships with other elements.

Why Learn Advanced Selectors?

✨ Benefits:

  • 🎯 Precise Targeting- Select exactly what you need
  • 🧹 Cleaner HTML- Less classes and IDs needed
  • ⚡ Dynamic- Automatically styles new content
  • 🎨 Interactive States- Hover, focus, active effects
  • 📦 Maintainable- Less code to manage

1. Attribute Selectors

Target elements based on their attributes and values.

Basic Attribute Selectors



/* Has attribute (any value) */
[disabled] {
    opacity: 0.5;
}

/* Exact match */
[type="text"] {
    border: 1px solid blue;
}

/* Specific class */
[class="button"] {
    padding: 10px;
}

Advanced Attribute Matching



/* Starts with */
[href^="https"] {
    color: green;
    /* Secure links */
}

/* Ends with */
[href$=".pdf"] {
    color: red;
    /* PDF links */
}

/* Contains */
[class*="btn"] {
    cursor: pointer;
    /* Any class with "btn" */
}

/* Word match (space-separated) */
[class~="active"] {
    font-weight: bold;
}

/* Starts with word (dash-separated) */
[lang|="en"] {
    /* Matches "en", "en-US", "en-GB" */
}

4. Combinators

Select elements based on their relationship with other elements. 



/* Descendant (space) - Any level deep */
div p {
    color: blue;
    /* All p inside div */
}

/* Child (>) - Direct children only */
div>p {
    color: red;
    /* Only direct p children of div */
}

/* Adjacent sibling (+) - Immediately after */
h2+p {
    font-weight: bold;
    /* First p after h2 */
}

/* General sibling (~) - All siblings after */
h2~p {
    color: gray;
    /* All p after h2 */
}

Selector Specificity

Understanding specificity helps you write more predictable CSS:

Selector Specificity Weight
Inline style 1, 0, 0, 0 Highest
ID 0, 1, 0, 0 High
Class, pseudo-class, attribute 0, 0, 1, 0 Medium
Element, pseudo-element 0, 0, 0, 1 Low 


/* Specificity examples */

/* 0,0,0,1 */
.class {}

/* 0,0,1,0 */
#id {}

/* 0,1,0,0 */
p.class {}

/* 0,0,1,1 */
div p.class {}

/* 0,0,1,2 */
#id .class p {}

/* 0,1,1,1 - Wins over 0,0,1,2 */
.class:hover {}

/* 0,0,2,0 */
div>p:first-child {}

/* 0,0,1,2 */

Best Practices

💡 Tips for Using Advanced Selectors:

  • Keep specificity low- Easier to override later
  • Use classes when appropriate- Don't over-complicate
  • Prefer :not() carefully- Can impact performance
  • Test pseudo-class support- Some are newer features
  • Use :is() to reduce repetition- More maintainable
  • Document complex selectors- Add comments explaining logic
  • Consider accessibility- :focus states are crucial
  • Avoid over-nesting- Max 3 levels deep

📚 Key Takeaways

  • Attribute selectors- Target by attributes: [href^="https"]
  • Pseudo-classes- Target states and positions: :hover, :first-child
  • Pseudo-elements- Style parts or add content: ::before, ::after
  • Combinators- Target by relationships: > , +, ~
  • :nth-child()- Powerful pattern matching for repeated elements
  • :not()- Exclude elements from selection
  • Form pseudo-classes- :valid, :invalid, :checked
  • Specificity matters- IDs>Classes>Elements
  • Double colon (::)for pseudo-elements, single (:) for pseudo-classes
  • Use selectors to reduce HTML clutter- less classes needed
  • Modern features- :is(), :where(), :has() are game-changers
  • Always ensure good browser supportfor production code
