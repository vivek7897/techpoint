Introduction to React

React is a popular JavaScript library for building user interfaces, particularly single-page applications. It was developed by Facebook and is maintained by Facebook and a community of developers.

What is React?

  • React is a JavaScript library for building user interfaces
  • It's component-based - build encapsulated components that manage their own state
  • React uses a Virtual DOM for efficient updates
  • It follows a declarative programming paradigm
  • Created by Facebook in 2011, open-sourced in 2013

Why Use React?

  • Component-Based: Build reusable UI components
  • Virtual DOM: Optimizes rendering and improves app performance
  • One-Way Data Binding: Provides better control over the application
  • JSX: Write HTML-like syntax in JavaScript
  • Large Ecosystem: Extensive third-party library support
  • Strong Community: Backed by Facebook and a large developer community

React Features

Key Features:

// 1. Components
function Welcome() {
    return <h1>Hello, React!</h1>;
}

// 2. JSX Syntax
const element = <div>Welcome to React</div>;

// 3. Virtual DOM
// React updates only the changed elements

// 4. One-way Data Flow
// Data flows from parent to child components

// 5. Hooks
import { useState, useEffect } from 'react';

React Use Cases

  • Single Page Applications (SPAs)
  • Mobile Applications (React Native)
  • Progressive Web Apps (PWAs)
  • Dashboard and Data Visualization Tools
  • Social Media Platforms
  • E-commerce Websites
💡 Tip: React is a library, not a framework. You have the flexibility to choose additional libraries for routing, state management, and more.

JSX - JavaScript XML

JSX stands for JavaScript XML. It allows you to write HTML-like code in JavaScript files. JSX makes React code more readable and expressive.

What is JSX?

JSX stands for JavaScript XML. It's a syntax extension that allows you to write \n HTML-like code directly in your JavaScript files. Even though it looks like HTML, it's actually \n JavaScript! React transforms JSX into regular JavaScript function calls using React.createElement().

Why Use JSX?

  • Familiar Syntax: If you know HTML, JSX is easy to learn
  • Visual Clarity: You can see the UI structure directly in your code
  • Better Error Messages: React shows helpful errors when something's wrong
  • Full JavaScript Power: Use any JavaScript expression inside JSX using {}
  • Type Safety: Works great with TypeScript for catching errors early

JSX Example:

// JSX Code
const element = <h1>Hello, World!</h1>;

// Without JSX (Pure JavaScript)
const element = React.createElement('h1', null, 'Hello, World!');

JSX Rules

  • Must return a single parent element
  • Use className instead of class
  • Close all tags (even self-closing ones)
  • Use camelCase for HTML attributes
  • JavaScript expressions go inside {}

Embedding Expressions

function Greeting() {
    const name = "John";
    const age = 25;
    
    return (
        <div>
            <h1>Hello, {name}!</h1>
            <p>You are {age} years old</p>
            <p>Next year you'll be {age + 1}</p>
        </div>
    );
}

React Components

Components are the building blocks of React applications. They let you split the UI into independent, reusable pieces.

Types of Components

React has two types of components: Functional Components and Class Components.

Functional Components

Simple Function Component:

// Basic functional component
function Welcome() {
    return <h1>Hello, World!</h1>;
}

// Arrow function component
const Welcome = () => {
    return <h1>Hello, World!</h1>;
};

// With props
function Greeting(props) {
    return <h1>Hello, {props.name}!</h1>;
}

// Usage
<Greeting name="John" />

Class Components

Class-based Component:

import React, { Component } from 'react';

class Welcome extends Component {
    render() {
        return <h1>Hello, World!</h1>;
    }
}

// With props
class Greeting extends Component {
    render() {
        return <h1>Hello, {this.props.name}!</h1>;
    }
}

// Usage
<Greeting name="John" />

Component Composition

function Header() {
    return <header><h1>My Website</h1></header>;
}

function Content() {
    return <main><p>Welcome to my site</p></main>;
}

function Footer() {
    return <footer><p>© 2026</p></footer>;
}

function App() {
    return (
        <div>
            <Header />
            <Content />
            <Footer />
        </div>
    );
}

State in React

State is a built-in object used to contain data that changes over time. Unlike props, state is managed within the component. Think of state as the component's memory.

What is State?

State is like a storage box inside your component. When data in the state changes, React automatically re-renders the component to show the new data. This is what makes React apps interactive!

Key Points About State:

  • State is private to the component
  • When state changes, component re-renders
  • State should not be modified directly
  • Use setState() or setters to update state
  • State updates may be asynchronous

State in Class Components

class Counter extends React.Component {
    constructor(props) {
        super(props);
        this.state = {
            count: 0
        };
    }
    
    increment = () => {
        this.setState({ count: this.state.count + 1 });
    }
    
    render() {
        return (
            <div>
                <p>Count: {this.state.count}</p>
                <button onClick={this.increment}>Increment</button>
            </div>
        );
    }
}

State in Functional Components (with Hooks)

import { useState } from 'react';

function Counter() {
    const [count, setCount] = useState(0);
    
    return (
        <div>
            <p>Count: {count}</p>
            <button onClick={() => setCount(count + 1)}>
                Increment
            </button>
        </div>
    );
}

Multiple State Variables

You Can Have Many States:

function UserProfile() {
    const [name, setName] = useState('John');
    const [age, setAge] = useState(25);
    const [isOnline, setIsOnline] = useState(true);
    
    return (
        <div>
            <h2>{name}</h2>
            <p>Age: {age}</p>
            <p>Status: {isOnline ? 'Online' : 'Offline'}</p>
            <button onClick={() => setAge(age + 1)}>Birthday</button>
            <button onClick={() => setIsOnline(!isOnline)}>Toggle Status</button>
        </div>
    );
}

State with Objects

Managing Complex State:

function UserForm() {
    const [user, setUser] = useState({
        name: '',
        email: '',
        age: 0
    });
    
    const updateName = (e) => {
        setUser({
            ...user,  // Keep other properties
            name: e.target.value  // Update only name
        });
    };
    
    return (
        <div>
            <input 
                value={user.name} 
                onChange={updateName}
                placeholder="Your name"
            />
            <p>Hello, {user.name}!</p>
        </div>
    );
}

Common State Mistakes

❌ Don't Do This:

// Wrong - Direct mutation
state.count = 5;

// Wrong - Modifying object directly
user.name = 'John';
setUser(user);

✅ Do This Instead:

// Correct - Use setter function
setCount(5);

// Correct - Create new object
setUser({ ...user, name: 'John' });
💡 Tip: State updates are asynchronous and may be batched for performance. Always use the previous state when the new state depends on the old state.

Event Handling in React

React events are similar to DOM events but with some key differences. React events use camelCase naming convention and are passed as functions. Events let your app respond to user actions like clicks, typing, scrolling, etc.

Why Event Handling?

Events make your app interactive. Without events, users can't interact with your app. Events tell React "Hey, the user clicked this button!" or "The user typed something!"

React Events vs HTML Events

HTML React
onclick onClick
onchange onChange
String function Function reference

Basic Event Handling

function Button() {
    const handleClick = () => {
        alert('Button clicked!');
    };
    
    return <button onClick={handleClick}>Click Me</button>;
}

// Inline function
<button onClick={() => alert('Clicked!')}>Click</button>

Event Object

function Form() {
    const handleSubmit = (e) => {
        e.preventDefault();
        console.log('Form submitted');
    };
    
    const handleChange = (e) => {
        console.log(e.target.value);
    };
    
    return (
        <form onSubmit={handleSubmit}>
            <input onChange={handleChange} />
            <button type="submit">Submit</button>
        </form>
    );
}

Common Events

Mouse Events:

  • onClick - When user clicks
  • onDoubleClick - When user double clicks
  • onMouseEnter - When mouse enters element
  • onMouseLeave - When mouse leaves element
  • onMouseMove - When mouse moves over element

Form Events:

  • onChange - When input value changes
  • onSubmit - When form is submitted
  • onFocus - When input gets focus
  • onBlur - When input loses focus

Keyboard Events:

  • onKeyDown - When key is pressed
  • onKeyUp - When key is released
  • onKeyPress - When key is pressed (deprecated)

Event with Parameters

function ItemList() {
    const handleDelete = (id) => {
        alert(\`Deleting item \${id}\`);
    };
    
    return (
        <div>
            {/* Method 1: Arrow function */}
            <button onClick={() => handleDelete(1)}>Delete Item 1</button>
            
            {/* Method 2: bind */}
            <button onClick={handleDelete.bind(null, 2)}>Delete Item 2</button>
        </div>
    );
}

Preventing Default Behavior

function LinkComponent() {
    const handleClick = (e) => {
        e.preventDefault();  // Stop default action
        console.log('Link clicked, but not navigating!');
    };
    
    return <a href="https://example.com" onClick={handleClick}>Click Me</a>;
}

Real Example: Interactive Button

function LikeButton() {
    const [likes, setLikes] = useState(0);
    const [isLiked, setIsLiked] = useState(false);
    
    const handleLike = () => {
        if (isLiked) {
            setLikes(likes - 1);
            setIsLiked(false);
        } else {
            setLikes(likes + 1);
            setIsLiked(true);
        }
    };
    
    return (
        <button 
            onClick={handleLike}
            style={{ backgroundColor: isLiked ? 'blue' : 'gray' }}
        >
            👍 {likes} {isLiked ? 'Liked' : 'Like'}
        </button>
    );
}
💡 Tip: React events are synthetic events - wrappers around native browser events that work consistently across all browsers. This means you don't have to worry about browser compatibility!

Conditional Rendering

In React, you can render different UI based on conditions using JavaScript operators. Think of it like IF-ELSE in your component - show this OR show that!

Why Conditional Rendering?

Sometimes you want to show different things based on certain conditions:

  • Show "Welcome Back!" if user is logged in, else show "Please Login"
  • Show loading spinner while data is loading
  • Show error message if something goes wrong
  • Hide/show parts of UI based on user permissions

If-Else Statement

function Greeting({ isLoggedIn }) {
    if (isLoggedIn) {
        return <h1>Welcome Back!</h1>;
    } else {
        return <h1>Please Sign In</h1>;
    }
}

Ternary Operator

function Greeting({ isLoggedIn }) {
    return (
        <div>
            {isLoggedIn ? <h1>Welcome!</h1> : <h1>Please Sign In</h1>}
        </div>
    );
}

Logical AND (&&)

function Mailbox({ unreadMessages }) {
    return (
        <div>
            <h1>Hello!</h1>
            {unreadMessages.length > 0 && (
                <h2>You have {unreadMessages.length} unread messages.</h2>
            )}
        </div>
    );
}

Multiple Conditions

More Complex Logic:

function UserStatus({ user }) {
    // Multiple conditions
    if (!user) {
        return <p>No user found</p>;
    }
    
    if (user.age < 18) {
        return <p>Access Denied: You must be 18+</p>;
    }
    
    if (!user.isPremium) {
        return (

                <p>Welcome, {user.name}!</p>
                <button>Upgrade to Premium</button>
            </div>
        );
    }

Switch Case Alternative

function StatusMessage({ status }) {
    const messages = {
        loading: '⏳ Loading...',
        success: '✅ Success!',
        error: '❌ Error occurred',
        idle: '💤 Waiting...'
    };

💡 Tip: Use the method that makes your code most readable. For simple conditions, use ternary or &&. For complex logic, use if-else or switch.

Lists and Keys

Use map() to render lists in React. Keys help React identify which items have changed, added, or removed. Think of keys as unique IDs for list items!

Why Lists?

In real apps, you often have arrays of data (users, products, comments) that you need to display. Instead of writing the same code 100 times, you use .map() to automatically create elements from your array!

Rendering Lists

function TodoList() {
    const todos = ['Learn React', 'Build Project', 'Deploy'];
    
    return (
        <ul>
            {todos.map((todo, index) => (
                <li key={index}>{todo}</li>
            ))}
        </ul>
    );
}

Keys

// Use unique IDs as keys
function UserList({ users }) {
    return (
        <ul>
            {users.map(user => (
                <li key={user.id}>
                    {user.name} - {user.email}
                </li>
            ))}
        </ul>
    );
}

Why Keys Are Important?

Keys tell React which item is which. Without keys, React can't tell if you added, removed, or reordered items. This can cause bugs and performance issues!

Forms in React

React forms work differently from HTML forms. Form data is handled by React components rather than the DOM. This gives you more control and makes form validation easier!

Why React Forms Are Different?

In HTML, forms maintain their own state. In React, we want React to be in charge of the state. This is called a "controlled component" - React controls the form!

Basic Form

import { useState } from 'react';

function MyForm() {
    const [name, setName] = useState('');
    
    const handleSubmit = (e) => {
        e.preventDefault();
        alert(`Hello, ${name}!`);
    };
    
    return (
        <form onSubmit={handleSubmit}>
            <input
                type="text"
                value={name}
                onChange={(e) => setName(e.target.value)}
            />
            <button type="submit">Submit</button>
        </form>
    );
}

Multiple Inputs

function SignupForm() {
    const [formData, setFormData] = useState({
        name: '',
        email: '',
        password: ''
    });
    
    const handleChange = (e) => {
        setFormData({
            ...formData,
            [e.target.name]: e.target.value
        });
    };
    
    return (
        <form>
            <input name="name" onChange={handleChange} />
            <input name="email" type="email" onChange={handleChange} />
            <input name="password" type="password" onChange={handleChange} />
        </form>
    );
}

Controlled Components

Controlled components have their value controlled by React state. React is the "single source of truth" for the input value.

What Are Controlled Components?

A controlled component is an input whose value is controlled by React state. Every time the user types, React updates the state, and that state controls what the input shows.

Benefits:

  • Easy to implement validation
  • You can disable submit button based on input
  • Enforce specific input formats
  • React has full control over the form

function ControlledInput() {

Forms Validation

Implement form validation to ensure data quality before submission. Validation helps users enter correct information and prevents errors.

Why Validation?

Validation ensures users enter the right data in the right format. It improves user experience by catching errors early and prevents bad data from reaching your server!

Types of Validation:

  • Required fields - Check if field is not empty
  • Format validation - Email, phone number format
  • Length validation - Min/max length
  • Pattern matching - Using regex
  • Custom validation - Business logic rules

Basic Validation:

function LoginForm() {
    const [email, setEmail] = useState('');
    const [password, setPassword] = useState('');
    const [errors, setErrors] = useState({});
    
    const validate = () => {
        const newErrors = {};
        
        // Email validation
        if (!email) {
            newErrors.email = 'Email is required';
        } else if (!/\S+@\S+\.\S+/.test(email)) {
            newErrors.email = 'Email is invalid';
        }
        
        // Password validation
        if (!password) {
            newErrors.password = 'Password is required';
        } else if (password.length < 6) {
            newErrors.password = 'Password must be at least 6 characters';
        }
        
        return newErrors;
    };
    
    const handleSubmit = (e) => {
        e.preventDefault();
        const validationErrors = validate();
        
        if (Object.keys(validationErrors).length === 0) {
            // No errors, submit form
            console.log('Form submitted successfully!');
        } else {
            // Show errors
            setErrors(validationErrors);
        }
    };
    
    return (
        <form onSubmit={handleSubmit}>
            <div>
                <input 
                    type="email"
                    value={email} 
                    onChange={e => setEmail(e.target.value)}
                    placeholder="Email"
                />
                {errors.email && <span style={{ color: 'red' }}>{errors.email}</span>}
            </div>
            
            <div>
                <input 
                    type="password"
                    value={password} 
                    onChange={e => setPassword(e.target.value)}
                    placeholder="Password"
                />
                {errors.password && <span style={{ color: 'red' }}>{errors.password}</span>}
            </div>
            
            <button type="submit">Login</button>
        </form>
    );
}

function RegistrationForm() {

        email: '',
        password: '',
        confirmPassword: ''
    });

        if (!formData.email) newErrors.email = 'Email is required';
        else if (!/\S+@\S+\.\S+/.test(formData.email)) {
            newErrors.email = 'Invalid email format';
        }
        
        if (!formData.password) newErrors.password = 'Password is required';
        else if (formData.password.length < 8) {
            newErrors.password = 'Password must be 8+ characters';
        }
        
        if (formData.password !== formData.confirmPassword) {
            newErrors.confirmPassword = 'Passwords do not match';
        }
        
        return newErrors;
    };
    
    const handleSubmit = (e) => {
        e.preventDefault();
        const validationErrors = validateAll();
        
        if (Object.keys(validationErrors).length === 0) {
            alert('Registration successful!');
        } else {
            setErrors(validationErrors);
        }
    };
    
    return (
        <form onSubmit={handleSubmit}>
            {Object.entries(formData).map(([key, value]) => (
                <div key={key}>
                    <input

                        value={value}
                        onChange={(e) => setFormData({...formData, [key]: e.target.value})}
                        placeholder={key}

                    {errors[key] && <span style={{ color: 'red' }}>{errors[key]}</span>}
                </div>

What is useState?

  • Multiple States: You can use useState multiple times in one component

useEffect Hook

useEffect lets you perform side effects in functional components like data fetching, subscriptions, or DOM manipulation.

What is useEffect?

useEffect is a Hook that lets you perform side effects in functional components. A "side effect" is anything that affects something outside the component - like fetching data from an API, updating the browser title, or setting up a timer.

What Are Side Effects?

Side effects are operations that interact with the outside world:

  • Fetching data from an API
  • Updating the document title
  • Setting up event listeners
  • Starting timers or intervals
  • Subscribing to external data sources
  • Manually changing the DOM

Why Use useEffect?

  • Lifecycle Replacement: Replaces componentDidMount, componentDidUpdate, componentWillUnmount
  • Run After Render: Code runs after component renders to screen
  • Control When It Runs: Use dependency array to control when effect runs
  • Cleanup: Return a cleanup function to prevent memory leaks

Basic Usage

import { useState, useEffect } from 'react';

function Example() {
    const [count, setCount] = useState(0);
    
    useEffect(() => {
        document.title = `Count: ${count}`;
    });
    
    return <button onClick={() => setCount(count + 1)}>Click</button>;
}

Dependency Array

// Runs only once (componentDidMount)
useEffect(() => {
    fetchData();
}, []);

// Runs when 'id' changes
useEffect(() => {
    fetchUser(id);
}, [id]);

// Runs on every render
useEffect(() => {
    console.log('Component rendered');
});

Cleanup Function

useEffect(() => {
    const subscription = props.source.subscribe();
    
    return () => {
        // Cleanup
        subscription.unsubscribe();
    };
}, [props.source]);

useRef Hook

useRef creates a mutable reference that persists across renders. Commonly used to access DOM elements directly.

What is useRef?

useRef is a Hook that creates a mutable reference that persists for the entire lifetime of the component. Think of it as a box where you can store any value that won't trigger a re-render when changed.

Two Main Uses:

  • No Re-render: Changing ref.current doesn't cause component to re-render
  • Persists Between Renders: Value stays the same across re-renders
  • Store Previous Values: Keep track of previous state or props

useContext Hook

useContext allows you to consume context values without wrapping components in a Consumer.

What is useContext?

useContext is a Hook that lets you read and subscribe to Context in your component. Context is a way to pass data through the component tree without having to pass props down manually at every level.

The Problem It Solves:

Imagine you have data (like user info, theme, language) that many components need. Without Context, you'd have to pass it through every component in between - this is called "prop drilling" and it's annoying! Context lets you "teleport" data directly to any component that needs it.

Why Use useContext?

  • Avoid Prop Drilling: No need to pass props through every component
  • Global State: Share data across the entire app (theme, user, language)
  • Cleaner Code: Simpler than passing props through many levels
  • Easy to Use: Much simpler than Context.Consumer

useMemo Hook

useMemo memoizes expensive calculations to optimize performance.

What is useMemo?

useMemo is a Hook that remembers (memoizes) the result of an expensive calculation. It only recalculates when one of its dependencies changes. Think of it as a smart cache for expensive operations!

How It Works:

When your component re-renders, useMemo checks if the dependencies changed. If they didn't, it returns the cached result instead of recalculating. This saves time and improves performance!

Why Use useMemo?

  • Performance: Avoid expensive calculations on every render
  • Optimization: Only recalculate when dependencies change
  • Better UX: Faster component renders = smoother app
  • Efficient: Don't waste CPU on unnecessary work

When to Use useMemo:

  • Expensive calculations (filtering large arrays, complex math)
  • Preventing unnecessary object/array recreation
  • Optimizing child components that depend on reference equality
import { useMemo } from 'react';

function ExpensiveComponent({ data }) {
    const expensiveResult = useMemo(() => {
        return data.reduce((sum, item) => sum + item, 0);
    }, [data]);
    
    return <div>Result: {expensiveResult}</div>;
}

useCallback Hook

useCallback memoizes callback functions to prevent unnecessary re-renders.

What is useCallback?

useCallback is a Hook that remembers (memoizes) a function. It returns the same function instance between renders unless dependencies change. This is useful when passing functions to child components.

The Problem:

In JavaScript, every time a component re-renders, all the functions inside it are recreated. Even if the function does the same thing, it's a new function object. This can cause child components to re-render unnecessarily.

Why Use useCallback?

  • Prevent Re-renders: Child components won't re-render if function doesn't change
  • Stable References: Function reference stays the same between renders
  • Performance: Reduces unnecessary work in React apps
  • With React.memo: Works great with memoized components

Difference from useMemo:

  • useMemo: Memoizes the result of a function
  • useCallback: Memoizes the function itself
import { useCallback } from 'react';

function Parent() {
    const handleClick = useCallback(() => {
        console.log('Clicked!');
    }, []);
    
    return <Child onClick={handleClick} />;
}

Custom Hooks

Custom Hooks let you extract component logic into reusable functions.

What Are Custom Hooks?

Custom Hooks are JavaScript functions that use React Hooks inside them. They let you extract component logic into reusable functions. Custom Hooks always start with "use" (like useCounter, useFetch, useLocalStorage).

Why Create Custom Hooks?

When you find yourself writing the same logic in multiple components, you can extract it into a Custom Hook. This makes your code DRY (Don't Repeat Yourself) and easier to maintain!

Benefits of Custom Hooks:

  • Reusability: Write once, use in many components
  • Organization: Keep related logic together
  • Easier Testing: Test logic separate from components
  • Share Logic: Share stateful logic between components
  • Cleaner Components: Components stay focused on UI

Rules for Custom Hooks:

  • Name must start with "use" (e.g., useCounter, useFetch)
  • Can call other Hooks inside them
  • Can return anything (values, functions, objects, arrays)
  • Follow the same rules as React Hooks
// Custom Hook
function useCounter(initialValue = 0) {
    const [count, setCount] = useState(initialValue);
    
    const increment = () => setCount(c => c + 1);
    const decrement = () => setCount(c => c - 1);
    const reset = () => setCount(initialValue);
    
    return { count, increment, decrement, reset };
}

// Usage
function Counter() {
    const { count, increment, decrement, reset } = useCounter(0);
    
    return (
        <div>
            <p>{count}</p>
            <button onClick={increment}>+</button>
            <button onClick={decrement}>-</button>
            <button onClick={reset}>Reset</button>
        </div>
    );
}

Redux

Redux is a predictable state container for JavaScript apps - the most popular state management library for React.

What is Redux?

Redux is a state management library that keeps all your app's state in one place (called the "store"). It uses a strict pattern: to change state, you dispatch an action, which goes through a reducer function that returns new state. This makes state changes predictable and traceable.

Why Use Redux?

  • Centralized State: All state in one place, easy to debug
  • Predictable: Same action always produces same result
  • Time Travel Debugging: Can replay actions and undo changes
  • Large Apps: Essential for complex apps with lots of shared state
  • DevTools: Amazing debugging tools to inspect every state change

Core Concepts:

  • Store: Single object that holds all application state
  • Actions: Plain objects describing what happened
  • Reducers: Pure functions that take old state and action, return new state
  • Dispatch: Send actions to reducers to update state

When to Use Redux:

  • Many components need the same data
  • State changes frequently
  • State updating logic is complex
  • Large codebase with many developers
// Actions
const increment = () => ({ type: 'INCREMENT' });
const decrement = () => ({ type: 'DECREMENT' });

// Reducer
function counterReducer(state = { count: 0 }, action) {
    switch (action.type) {
        case 'INCREMENT':
            return { count: state.count + 1 };
        case 'DECREMENT':
            return { count: state.count - 1 };
        default:
            return state;
    }
}

// Store
import { createStore } from 'redux';
const store = createStore(counterReducer);

What Are API Calls?

API calls are requests your React app makes to a server to get or send data. Almost every real application needs to fetch data from an API (like user data, posts, products) or send data to a server (like submitting forms, uploading files).

Manage loading states to provide better user experience while data is being fetched.

What Are Loading States?

Loading states tell users that something is happening when data is being fetched from an API. Instead of showing a blank screen or old data, you show a loading spinner, skeleton screen, or "Loading..." message. This improves user experience significantly!

Why Use Loading States?

  • Less Confusion: Clear feedback prevents users from clicking multiple times
  • Professional: Makes your app feel polished and well-made
  • Perceived Performance: App feels faster when you show progress

Types of Loading Indicators:

  • Spinners: Circular loading animations
  • Progress Bars: Show percentage of completion
  • Skeleton Screens: Gray boxes that mimic content layout
  • Text Messages: Simple "Loading..." text
function App() {
    const [loading, setLoading] = useState(true);
    const [data, setData] = useState(null);
    
    useEffect(() => {
        setLoading(true);

Performance Optimization

Techniques to optimize React application performance. A fast app = happy users!

Why Performance Matters?

Slow apps drive users away! Performance optimization makes your app load faster, respond quicker, and use less memory. This means better user experience and higher conversion rates.

Common Performance Issues

  • Unnecessary re-renders
  • Large bundle sizes
  • Heavy computations on every render
  • Loading everything at once
  • Not caching expensive operations

Optimization Techniques

1. Use React.memo for Component Memoization

// Without memo - re-renders every time parent renders
function ExpensiveComponent({ data }) {
    console.log('Rendering...');
    return <div>{data}</div>;
}

// With memo - only re-renders when props change
const OptimizedComponent = React.memo(function ExpensiveComponent({ data }) {
    console.log('Rendering...');
    return <div>{data}</div>;
});

2. Use useMemo for Expensive Calculations

function DataProcessor({ items }) {
    // Without useMemo - recalculates on every render
    const processedData = items.map(item => expensiveOperation(item));
    
    // With useMemo - only recalculates when items change
    const memoizedData = useMemo(() => {
        return items.map(item => expensiveOperation(item));
    }, [items]);
    
    return <div>{memoizedData.length} items</div>;
}

3. Use useCallback for Function Memoization

function Parent() {
    // Without useCallback - new function on every render
    const handleClick = () => {
        console.log('Clicked');
    };
    
    // With useCallback - same function reference
    const memoizedCallback = useCallback(() => {
        console.log('Clicked');
    }, []);
    
    return <Child onClick={memoizedCallback} />;
}

4. Virtualize Long Lists

// Only render visible items, not all 10,000
import { FixedSizeList } from 'react-window';

function LongList({ items }) {
    return (
        <FixedSizeList
            height={500}
            itemCount={items.length}
            itemSize={35}
            width="100%"
        >
            {({ index, style }) => (
                <div style={style}>{items[index]}</div>
            )}
        </FixedSizeList>
    );
}

5. Code Splitting with Dynamic Imports

// Load component only when needed
const HeavyComponent = lazy(() => import('./HeavyComponent'));

function App() {
    return (
        <Suspense fallback={<div>Loading...</div>}>
            <HeavyComponent />
        </Suspense>
    );
}

6. Optimize Images and Assets

  • Use WebP format for images
  • Lazy load images below the fold
  • Use image CDN for optimization
  • Compress images before upload
  • Use appropriate image sizes

Error Boundaries

Error boundaries catch JavaScript errors in component tree and display fallback UI.

class ErrorBoundary extends React.Component {
    state = { hasError: false };
    
    static getDerivedStateFromError(error) {
        return { hasError: true };
    }
    
    componentDidCatch(error, errorInfo) {
        console.log(error, errorInfo);
    }
    
    render() {
        if (this.state.hasError) {
            return <h1>Something went wrong.</h1>;
        }
        return this.props.children;
    }
}

// Usage
<ErrorBoundary><App /></ErrorBoundary>
